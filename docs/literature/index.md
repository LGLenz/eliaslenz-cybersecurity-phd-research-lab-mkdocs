# Literature review

The literature review weaves together several strands:

- cloud security and assurance for PaaS/IaaS
- governance, risk and compliance (GRC) practices
- policy-as-code (PaC) and infrastructure-as-code (IaC)
- human factors in security and compliance
- explainability and transparency in automated decision systems

The review is supported by:

- structured mapping tables (CSV) for key publications
- curated reference collections (BibTeX / Markdown)
- early synthesis work feeding into the CPS cybersecurity preprint

## Objectives

The objectives of the review are to identify:

- where PaC is used mainly as a tooling convenience, with limited governance
  integration
- where governance frameworks lack operationalised, testable artefacts
- patterns for making complex controls understandable to non-specialists
